# LinkedIn Post Writing Conventions

This skill provides voice and style conventions to follow when writing or editing LinkedIn posts for blog projects.

## Style Guide

Read the full style guide at: `library/creative-dna/writing/linkedin-post-style-guide.md`

Key rules:
- **Post is the trailer, not the movie** — Drive traffic to the blog and demo; never replicate the full content
- **Hook first** — First 2-3 lines must create pull before "see more" truncation. Never start with "I'm excited to share..."
- **Named pattern** — Every Build Post names the pattern (e.g., "3 Amigo Agents", "Cross-Device Orchestration")
- **Exact numbers** — "21 SQL statements" not "lots of database work". Specificity is credibility
- **Closing signature line** — Single memorable line connecting back to real life, not a technical summary
- **Resource block** — Always end with blog link + demo video link (emoji-prefixed)
- **4-8 specific hashtags** — Descriptive, not vibes (#AIAgents not #AI)

## Post Categories

Choose the right category before writing:

| Category | Use When |
|---|---|
| A: Build Post | Showcasing a project — most common for blog announcements |
| B: Tool Discovery | Reacting to a tool or feature that changed the workflow |
| C: Industry Insight | Connecting news/trends to George's own experience |
| D: Life + AI | Personal problem solved with AI — most engaging hook |
| E: Milestone/Announcement | Launch, transition, or career update — use sparingly |

## Build Post Structure (most common for blog announcements)

```
[HOOK — 1-3 lines]
Personal moment, bold claim, or specific timestamp. Never generic.

[CONTEXT — 2-4 lines]
What was built, what problem it solves, why it matters.

[PROOF — emoji-bulleted details]
💡 Key technical detail
💡 Key technical detail
⚠️ Caveat or "here's the rub" (optional)

[INSIGHT — 2-3 lines]
Name the pattern. "What this really means..." or "The key insight..."

[HUMAN LANDING — 1 line]
The memorable closing. Real life, not technical.

[RESOURCES]
📝 Blog link
🎥 Demo video link
💻 Code repo link (if public)

[HASHTAGS]
#Specific #Relevant #Tags
```

## Context to Read Before Writing

1. The finished blog markdown (understand the full content — the post links to it)
2. The demo video URL (the proof — goes in the resource block)
3. The gold outline (understand the narrative arc and the named pattern)

## DO NOT

- Start with "I'm excited to share..." or "In today's world..."
- Write pure opinion without a build or project behind it
- Use vague quantifiers ("saved a lot of time" → use exact: "Days of slide-building, one afternoon")
- Reproduce the blog content — link to it instead
- Use engagement-bait questions ("What do you think? Drop a comment!")
- Stack emojis (🎉🔥💪) or use generic hashtags like #AI #Innovation
- Skip the named pattern or the closing signature line
- Forget the resource block with blog + video links
