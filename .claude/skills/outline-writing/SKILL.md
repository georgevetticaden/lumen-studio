## Presentation Outline Standards

Read the full style guide: library/creative-dna/writing/presentation-deck-style-guide.md

### Quick Reference
- Philosophy: Build the story first — slides serve the story, never the reverse
- Narrative arc: Define as a phrase chain FIRST (e.g., Hook → Teach → Feel → Equip → Send)
- Flow summary: ASCII journey diagram showing descent → turn → ascent
- Foundation before feelings: Build understanding before asking for emotional engagement
- Every slide gets: title, subtitle, builds, time estimate, key points, transition

### Slide Composition Patterns
Use these named patterns when specifying slide content in the outline:
- **A: Title Slide** — centered, generous negative space, no content
- **B: Section Divider** — minimal, signals major transition
- **C: Concept + Evidence** — claim + data/quote + image + key insight
- **D: Card Grid** — 2-4 parallel concepts in horizontal cards
- **E: Visual Diagram** — cycle, before/after, journey timeline
- **F: Repeating Framework** — identical layout for each element (George's signature)
- **G: Recap/Mapping** — maps example back to framework after film/demo
- **H: Activity/Discussion** — clear question + 3 numbered steps + time
- **I: Q&A** — pre-loaded or live, direct and warm answers
- **J: Climax** — most restrained slide, maximum negative space, one powerful element

### Content Design Patterns
- Subtitle as compass — every slide has one, tells audience how to receive the slide
- The Reframe — "What [X] is really saying" in plain modern language
- Tag pattern — 3-4 short tags grounding abstract concepts in modern experience
- Transition statements — every major slide bridges to the next section
- Progress indicators — "N of M" on repeating framework series
- Authority + personal experience pairing

### Outline Document Format
- Version in filename: {topic}-outline-v1.md ... vN.md
- Header: title, presenter, audience, time, date
- Narrative arc section FIRST
- Flow summary as ASCII diagram
- Complete slide list table
- Per-slide specs: file, builds, time, content, key points, transition
- Timing table that adds up
- Status emoji per slide: ✅ No change, 🆕 New, 🔄 Updated

### Timing Guidelines
- 45-60 min presentation: 14-18 slides, ~3-4 min avg
- 60-75 min presentation: 16-24 slides, ~3-4 min avg
- Hook: 5-8 min | Foundation: 15-25 min | Activity: 10-18 min
- Film/Demo: 5-15 min | Climax: 5-6 min | Closing: 3-5 min
